UX Strategy & Conversion Assessment

Le Petit Croissant User Journey Audit

Prepared for: Management Team
Date: February 2026

Executive Summary

Le Petit Croissant has strong upper-funnel brand appeal, but requires immediate intervention to remove lower-funnel friction blocking conversion

1. Situation

The current website effectively communicates brand premiumness through high-quality visual assets and clear value propositions.

  • Strong aesthetic alignment
  • Clear B2B positioning
  • Mobile-responsive foundation

2. Complication

The ordering journey lacks cohesive flow and suffers from critical usability breaks. A rigid "Gatekeeper Modal" blocks entry, while broken product interactions, hidden inventory limits, and disorienting page reloads create severe user frustration.

  • 6 mandatory clicks to begin shopping
  • Products lack clickable detail views
  • Disorienting post-form redirection
  • Hidden location-based stock rules

3. Resolution

Complete structural overhaul of the user journey: Defer logistics to checkout, mandate global inventory transparency upfront, and fix broken UI links to restore user trust and cart momentum.

  • Defer fulfillment mapping
  • Enable rich product modals
  • Ensure linear journey routing
Current State Assessment

The current user flow introduces a critical "choke point" at the exact moment a customer exhibits purchase intent

Symptom Diagnosis

  • False "Dead Ends" (Modal Friction): Product modals exist, but many require mandatory checkboxes (e.g., "48h pre-order"). If a user misses this and clicks "Add to Cart", a tiny red text prompt appears, making the button feel broken or unresponsive.
  • Disorienting Routing: After enduring the 6-click location modal, the system abruptly reloads the entire page from the top, forcing the user to re-scroll and re-orient themselves back to the menu.
  • Volatile Cart State: Interacting with the location/delivery method header mid-session aggressively clears the user's cart without adequate warning, destroying built purchase momentum.

Business Impact

  • Eroded Confidence: Broken clicks and confusing reloads make the site feel untrustworthy for high-value corporate catering orders.
  • Guaranteed Abandonment: Late-stage "out of stock at location" errors trigger immediate rage-quits. The user journey lacks cohesive logical steps.
Deep Dive: The "Gatekeeper" Modal

The fulfillment selection mechanism requires six distinct actions, acting as a barrier to initial product engagement

The Anatomy of Friction:

  1. User clicks "Order Now"
  2. Decision: Select "Pickup" or "Delivery"
  3. User waits for map rendering
  4. Action: User must locate and click radio button for specific address
  5. Action: Click "Next"
  6. Action: Click "Save"

Strategic Imperative: Best-in-class e-commerce architecture (e.g., modern Shopify/Wix implementations) pushes logistical framing to the final checkout stage or handles it via lightweight geolocation prompts.

Strategic Recommendations

We recommend prioritizing four core structural fixes to establish a linear, trusting user journey

Initiative Proposed Future State Impact Effort / Feasibility
1. Fulfillment Deferral
(Priority 1)		 Remove the pre-browse fulfillment modal. Allow users to immediately select "Add to Cart" directly from the product grid. Prompt for delivery/pickup details only upon clicking "Checkout". High Medium
(Requires CMS/Cart logic adjustment)
2. Fix Product Anchoring & Modals
(Priority 1)		 Make mandatory modal fields (like pre-order acknowledgements) highly visible. Ensure the post-location form submission smoothly scrolls the user *down* to the menu grid, never reloading the page or losing state. High Low
(Basic UI routing fix)
3. Cart Persistence & Filtering Never clear a user's cart when they modify delivery logic unless absolutely necessary. Instead, flag incompatible items *within* the cart for removal. Ask for location upfront using a lightweight dropdown. High Medium-High
(Inventory logic integration)
4. Checkout Refinement Remove or minimize the default "Tip" percentage prompt in the B2B/Catering context, as it introduces unexpected friction for corporate buyers. Low-Med Low
(Checkout settings toggle)
Path Forward

Implementing the "Fulfillment Deferral" initiative will immediately reduce drop-off and tangibly scale conversion rates

Next Steps

  • Engage technical lead/agency to review Wix/CMS cart logic constraints regarding deferred fulfillment.
  • Implement UI Hierarchy Realignment (Homepage CTA & Grid text truncation) within the current sprint cycle.
  • Deploy A/B testing on the checkout flow to monitor the impact of removing tipping terminology for corporate buyers.
